The AI Token Budget

AI adoption is becoming a token-allocation problem. The winners will not merely buy access to better models. They will know which workflows deserve expensive inference, which ones need cheaper intelligence, and who is accountable when agents spend compute on behalf of the business.

The next serious AI budget fight will sound boring. It will not begin with a philosophical argument about intelligence. It will begin with a line item: token consumption.

That is a healthy development. For two years, many executives treated tokens as a technical abstraction, something buried inside a vendor invoice or hidden by a flat software license. But inference is where AI becomes operating cost. Training produces capability. Inference spends it. Every prompt, tool call, retrieved document, chain-of-thought substitute, image frame, voice turn, planning loop, and agent retry converts strategy into consumption.

This is why the current infrastructure debate matters to operators. McKinsey's June 2026 analysis argues that cost per token and energy per token are becoming defining measures of AI economics. Goldman Sachs Research expects agentic AI to drive a 24-fold increase in token consumption by 2030, while Goldman Sachs Asset Management says enterprise token use is already widening between heavy adopters and median companies. NIST and CAISI, meanwhile, are pushing agent standards and identity work because agents need access to real systems before they create real value.

The management problem sits at the intersection of those signals. If agents work, token volume rises. If token volume rises, cost and energy discipline matter. If agents touch real systems, identity and authorization matter. The budget cannot stay inside engineering.

The token budget is the place where AI ambition meets operating discipline.

The Old Budget Was Easy

The old AI budget was mostly a procurement question. Which model? Which copilot? Which seat count? Which vendor contract? The spend looked like software, so it was governed like software.

That worked while usage was light, experimental, or bounded by human prompting. A team could buy a tool, run pilots, and accept a fuzzy relationship between activity and value. The main question was whether people adopted it.

Agentic systems change the shape of the cost. A human asks for an outcome. The system may plan, call tools, search files, inspect databases, write drafts, check its work, ask another model, retry a failed path, and generate logs. The user sees one answer. The budget sees a chain of inference events.

That gap creates managerial risk. The employee thinks they requested a task. The system consumed a budget. The vendor sees usage. The finance team sees a bill. The security team sees a new actor touching systems. The business still has to decide whether the work was worth doing.

The Unit Has Changed

Seat pricing taught managers to think in users. Token pricing forces them to think in work.

A user can be cheap and wasteful, or expensive and valuable. A token can be spent on a hallucinated summary, a useful code review, a low-value slide rewrite, a fraud triage event, a customer support resolution, or an agent loop that should have stopped five calls earlier. The unit that matters is not the token itself. It is the priced unit of judgment, coordination, generation, or action that the token makes possible.

That distinction is why blanket AI rationing is crude. A company can save money by forcing every workflow onto a small model, then quietly lose the economic value that justified AI in the first place. It can also waste money by running frontier inference on work where a smaller model, deterministic script, search index, template, or human decision would do.

The goal is not fewer tokens. The goal is tokens that buy the right kind of advantage.

The Budget Map

Executives need a map before they need a dashboard. A useful token budget has five lines.

Budget Line Management Question Failure Mode
Model tier Which tasks require frontier inference, and which can run on cheaper models or deterministic tools? Using premium intelligence as a default setting.
Context budget How much retrieved or pasted context is worth paying for before the marginal document adds noise? Long prompts that feel thorough but inflate cost and error surface.
Agent loop How many planning, tool-use, retry, and verification steps are allowed before escalation? Autonomy that spends silently because nobody set a stop rule.
Value class Is this inference buying revenue, risk reduction, cycle-time compression, learning speed, or convenience? Counting every AI-assisted artifact as equally valuable.
Owner Who owns the bill, the workflow outcome, the identity permissions, and the rollback trigger? A shared budget with no responsible operator.

This map turns token spend into a management artifact. Model tier separates expensive cognition from routine work. Context budget prevents the organization from paying to shovel documents into prompts without relevance discipline. Agent loop makes autonomy accountable. Value class keeps convenience from masquerading as strategy. Owner prevents compute from becoming an orphan cost.

The Cheap Token Trap

Falling token costs will not remove the need for budgeting. They may make it more important.

McKinsey identifies model optimization, advanced packaging, custom silicon, and co-packaged optics as major levers for reducing inference cost over time. If these improvements arrive, more workflows will become economically plausible. That is good news. It also creates a familiar trap: when the unit gets cheaper, usage expands faster than discipline.

A cheaper token can still be wasted. A cheap agent loop can still touch the wrong system. A cheap summary can still mislead a decision. A cheap retrieval process can still expose sensitive context. A cheap workflow can still create review burden for humans who now have to inspect more machine output.

The lesson is not austerity. It is that cost decline should widen the test set, not suspend judgment. As tokens get cheaper, the question shifts from "Can we afford to run this?" to "Should this workflow be allowed to consume intelligence automatically?"

The Security Line

The token budget also needs a security line because agents do not merely answer. They act.

NIST's AI Agent Standards Initiative frames the adoption problem around trust, interoperability, security, and identity. Its RFI summary says respondents widely agreed that AI agents create novel security threats and that those concerns are a barrier to adoption. That belongs in the budget discussion, not only in a security appendix.

Once an agent has permissions, spending tokens can mean spending authority. It may read a customer record, update a CRM field, trigger a support workflow, call an internal API, draft an email, or query a privileged database. Each step has an inference cost and an authorization cost. The company should not optimize one while ignoring the other.

For serious workflows, the token budget should be paired with identity controls: named human owner, scoped permissions, just-enough access, audit trail, step limit, exception path, and rollback rule. Otherwise the company is not budgeting AI. It is giving a probabilistic worker a company card.

The Founder Version

A founder does not need an enterprise FinOps team to start. They need three rules.

  1. Put every AI workflow into a value class before scaling it: revenue, delivery, risk, learning, or convenience.
  2. Set a maximum loop depth for agentic work: number of tool calls, retries, documents, or minutes before human review.
  3. Review one weekly token ledger: top workflows by spend, output produced, human review burden, and stop-rule exceptions.

This is enough to prevent the two founder mistakes: starving a valuable workflow because the bill feels new, and subsidizing an impressive workflow because the output feels magical. The test is simple. If a workflow cannot name what business metric token spend improves, it stays experimental.

The CFO Question

The CFO should not ask for a single AI cost number. That number hides the decision.

Ask instead: which token spend is strategic, which is operating leverage, which is employee convenience, and which is waste? A coding assistant that reduces review bottlenecks may deserve more expensive inference than a presentation assistant. A fraud agent may justify heavy context retrieval if it reduces loss or escalation. A meeting-summary tool may need a hard ceiling because the upside is modest and the volume is large.

The CFO's job is not to suppress consumption. It is to price autonomy correctly. The budget should reward high-value inference and force low-value workflows to simplify, downshift, cache, template, or stop.

The Executive Test

Before approving a large AI workflow, require six answers.

  1. Which model tier is the default, and what event allows escalation?
  2. What context is allowed into the prompt or retrieval path?
  3. How many agent steps are permitted before a human owns the decision?
  4. What business value class justifies the token spend?
  5. Which human owns the bill, permissions, and rollback?
  6. What weekly report would show that the workflow should be expanded, downshifted, or killed?

These questions make AI spending legible. They also make strategy less theatrical. A company can say yes to more AI while still saying no to unmanaged inference.

The Better Argument

The market argument will keep swinging between abundance and scarcity. One side will point to falling cost per token. The other will point to infrastructure strain, energy limits, chip shortages, and rising agent usage. Both can be right.

The operator's argument is narrower. AI will be cheap enough to enter many workflows and expensive enough to punish lazy design. That combination rewards companies that can allocate tokens like capital: deliberately, observably, and with a theory of return.

The companies that win the next phase will not ask only whether AI works. They will ask where intelligence should be spent.
